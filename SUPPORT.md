# Support

## Supported Topics

AeroBIM supports public engineering discussion for:

1. reproducible BIM validation bugs;
2. API/report contract regressions;
3. deterministic pipeline behavior and provenance tracking issues;
4. documentation and governance corrections;
5. benchmark and quality-gate reproducibility problems.

## Not a Support Channel

1. private vulnerability disclosure in public issues;
2. emergency production support commitments;
3. confidential project data handling via public channels;
4. open-ended consulting requests without reproducible technical scope.

## Before Opening a Request

Read:

1. README.md
2. CONTRIBUTING.md
3. SECURITY.md
4. docs/15-local-quality-gate.md

## Public Channels

1. Bug reports: GitHub Issues templates.
2. Feature proposals: GitHub Issues templates.
3. Code changes: Pull Requests with validation evidence.

## Response Model

AeroBIM follows best-effort maintainer triage for open-source support.
